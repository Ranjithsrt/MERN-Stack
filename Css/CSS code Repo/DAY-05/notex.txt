CSS Position Property
1. static (Default)
Behavior: Elements are positioned according to the normal document flow.​

Limitations: top, right, bottom, and left properties have no effect.​

2. relative
Behavior: Elements are positioned relative to their normal position.​

Capabilities: top, right, bottom, and left properties move the element from its normal position without affecting surrounding elements.​

3. absolute
Behavior: Elements are removed from the normal document flow and positioned relative to the nearest positioned ancestor (an ancestor with a position other than static).​

Capabilities: top, right, bottom, and left properties specify the exact position. Surrounding elements ignore the space the element would have occupied.​

4. fixed
Behavior: Elements are positioned relative to the viewport; they remain in the same position regardless of scrolling.​

Use Cases: Commonly used for persistent headers, footers, or sidebars that remain visible during scrolling.​

5. sticky
Behavior: Elements act like relatively positioned elements until they reach a specified scroll position, then they become fixed.​
